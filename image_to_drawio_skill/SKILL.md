# 任意图片转可编辑 draw.io 文件 Skill

## 目标

把任意一张扁平图片、截图、信息图、流程图、架构图、海报、界面图、手绘草图或插画，重建为可编辑的多图层 draw.io / diagrams.net 文件，并同时输出可复用素材包。

核心原则：能用 draw.io 原生对象重画的内容，优先使用原生对象；结构清晰的小元素优先做成独立 SVG；复杂背景、照片、纹理和高频细节保留为 PNG。目标是获得“可继续编辑、结构清楚、视觉接近原图”的源文件。

## 适用场景

当用户上传一张图片，并希望得到以下结果时使用：

- 可编辑的 `.drawio` 文件；
- 分层素材包；
- 背景、主体、线条、文字、标注分离；
- 文字、箭头、连接线、标签框可在 draw.io 中直接修改；
- 图标、按钮、简单对象可缩放、改色、复用；
- 复杂区域以 PNG 保留，降低重建失真。

常见输入包括截图、流程图、组织结构图、网页或 App 界面、课程图示、技术架构图、产品说明图、地图示意图、宣传海报、图表截图、白板照片和手绘草图。

## 不适用边界

以下内容不建议强行完全矢量化：

- 写实照片；
- 复杂自然纹理；
- 密集渐变、阴影和光效；
- 大面积手绘笔触；
- 模糊、低分辨率或严重压缩的图片；
- 包含大量小字号文字且无法准确识别的截图。

这些区域应作为背景 PNG 或局部 PNG 处理。需要像素级一致时，保留原图或修复后的背景图；需要可编辑时，重画关键结构和文字。

## 输入

用户通常会提供：

1. 一张原始图片；
2. 输出要求，例如只要 draw.io、需要素材包、需要预览图、需要保持原尺寸等；
3. 可选修改要求，例如替换文字、统一字体、调整配色、删除水印、重排结构、改成中文标签、保留原风格等。

若用户没有特别说明，默认保留原图比例和主要视觉风格，尽量让文字、线条、箭头、框、图例和主要对象可编辑。

## 输出

建议输出一个 zip 包，包含：

```text
project_name_drawio_package/
├── project_name.drawio                 # 主文件，可用 draw.io / diagrams.net 打开编辑
├── preview.png                         # 重建后的预览图或截图
├── assets/
│   ├── background.png                  # 复杂背景或底图
│   ├── object_*.svg                    # 独立对象、图标、按钮、卡片等
│   ├── region_*.png                    # 不适合矢量化的局部复杂区域
│   └── effect_*.svg                    # 简单装饰、强调符号、视觉效果
├── manifest.json                       # 元素清单、坐标、图层、样式说明
├── README_使用说明.md                  # 打开、编辑、替换素材说明
└── source_analysis.md                  # 原图元素分析与重建策略
```

## 工作流程

### 1. 读取并分析原图

先记录图片的宽高、比例、主体布局和视觉风格。再把图片拆成以下层次：

- 背景层：照片、纹理、渐变、环境、底图、装饰底板；
- 容器层：页面框架、分栏、卡片、面板、表格、泳道、模块框；
- 主体层：图标、按钮、人物、物体、设备、建筑、界面组件、流程节点；
- 连接层：实线、虚线、箭头、曲线、折线、流程连接、指示线；
- 数据层：图表、坐标轴、柱形、折线、饼图、仪表盘、数值标签；
- 标注层：标题、正文、注释、图例、编号、说明框、气泡框；
- 效果层：阴影、高光、半透明遮罩、强调圈、装饰符号。

分析时记录：

- 每个元素的大致位置、尺寸、颜色、透明度；
- 元素之间的遮挡关系；
- 哪些部分适合做 PNG，哪些适合做 SVG，哪些适合用 draw.io 原生形状；
- 文字内容是否需要 OCR、人工校对或按用户要求改写；
- 哪些元素需要锁定，哪些元素需要保持可编辑；
- 是否存在旧内容、水印、噪声、错字或用户要求删除的内容。

### 2. 确定重建策略

按可编辑性优先级处理：

1. 文字：优先转为 draw.io 文本对象；
2. 线条和箭头：优先转为 draw.io 连接线；
3. 矩形、圆角矩形、圆、菱形、卡片、表格：优先转为 draw.io 原生形状；
4. 简单图标和清晰对象：优先转为独立 SVG；
5. 复杂照片、纹理、阴影和手绘区域：保留为 PNG；
6. 难以重建但视觉重要的局部区域：裁切为局部 PNG。

不要把整张图片直接作为唯一背景嵌入，除非用户明确只需要低成本的 draw.io 容器文件。通用目标是尽可能拆分为可编辑对象。

### 3. 生成或修复背景 PNG

复杂背景不要拆成大量矢量对象。推荐生成一张干净底图：

- 移除前景文字、箭头、图标、按钮、框线、图例等信息元素；
- 保留原图的比例、透视、色调、光照、风格和构图；
- 对被移除区域进行自然补全；
- 输出尺寸与原图一致，或按用户指定尺寸输出。

背景图命名为：

```text
assets/background.png
```

### 4. 生成 SVG 素材

适合 SVG 化的元素包括：

- 图标、符号、按钮、Logo 的简化版本；
- 边界清晰的对象；
- 简单插画元素；
- 可复用的装饰图形；
- 需要后续改色或缩放的元素。

SVG 要求：

- 使用 `viewBox`；
- 优先使用 `path`、`rect`、`circle`、`ellipse`、`polygon`、`line`、`polyline`；
- 避免外链字体、脚本、复杂滤镜和嵌套大位图；
- 每个元素独立成文件；
- 文件名语义化，例如 `primary_icon.svg`、`user_avatar.svg`、`chart_marker.svg`、`warning_symbol.svg`。

### 5. 使用 draw.io 原生对象重建可编辑内容

以下内容尽量用 draw.io 原生对象：

- 标题、正文、图例文字、编号；
- 矩形、圆角矩形、圆形、菱形、分组框；
- 表格边框、卡片、面板、泳道；
- 箭头、引线、折线、曲线、虚线；
- 简单图表中的柱、线、点、坐标轴；
- 简单图标的几何重画版本。

文字处理规则：

- 能识别的文字要重录为可编辑文本；
- OCR 结果必须校对，尤其是数字、单位、专有名词、大小写；
- 保留原字号层级、粗细、颜色和对齐关系；
- 中文默认可使用宋体、黑体或用户指定字体；
- 英文默认使用常见无衬线字体，除非原图风格要求其他字体。

### 7. 组合 draw.io 文件

组合顺序建议：

1. 背景 PNG；
2. 容器、卡片、分栏、表格底框；
3. 大型主体对象；
4. 连接线、箭头、流程线；
5. 图表元素、数据标记；
6. 小图标和装饰元素；
7. 文本、标注、图例；
8. 说明层和辅助层。

推荐图层：

- `00_background`
- `10_containers`
- `20_objects`
- `30_connectors`
- `40_data`
- `50_labels`
- `60_effects`
- `70_notes`

### 8. 导出预览并自检

生成 `.drawio` 后必须导出预览图或截图检查。重点看：

- 画布比例是否正确；
- 背景是否拉伸变形；
- 关键元素是否对齐；
- 文字是否准确、清晰、可编辑；
- 线条、箭头和连接关系是否正确；
- 层级和遮挡关系是否合理；
- SVG 是否正常显示；
- draw.io 打开后是否能直接编辑主要元素；
- 打包文件是否完整。

发现问题后，修改 `manifest.json`、SVG 或 PNG，再重新生成。

## 图层判断规则

### 使用 PNG 的情况

- 大面积照片、纹理、渐变、插画背景；
- 难以稳定矢量化的复杂局部；
- 被前景元素遮挡后需要补全的区域；
- 用户更重视视觉一致性，编辑需求较低的区域。

### 使用 SVG 的情况

- 边界清晰、结构简单、可复用的对象；
- 图标、符号、按钮、简单插画；
- 需要缩放、改色、单独替换的元素。

### 使用 draw.io 原生对象的情况

- 文字；
- 矩形、圆角矩形、圆、菱形、标签框；
- 箭头、线段、曲线、虚线；
- 表格、泳道、流程节点；
- 简单图表和简单几何图形。

## 推荐质量标准

- 尽量保持原图比例、布局、颜色和视觉层级；
- 主要文字全部可编辑；
- 线条、箭头、框线可编辑；
- 主要对象分层清楚；
- 复杂背景不失真；
- SVG 文件可独立复用；
- 文件命名清楚；
- README 写明打开方式、编辑方式、素材替换方式；
- manifest 能解释每个元素的位置、图层和用途。

## 常见问题

### 背景生成后与原图不完全一致

这是正常情况。复杂背景的目标是提供干净画布和接近原图的视觉风格。若用户要求像素级一致，应在原图上做局部修复，减少重新生成。

### OCR 文字不准确

必须人工校对。数字、单位、大小写、标点和专有名词优先检查。无法确认的文字应在 `source_analysis.md` 中标注。

### SVG 无法在 draw.io 中显示

检查 SVG 是否包含不兼容标签。尽量避免外链字体、脚本、复杂滤镜和嵌套位图。必要时简化为基础 path 或改用 PNG。

### draw.io 文件打开后元素不可编辑

通常是因为元素被整张图嵌入。应将文字、箭头、框线、流程节点改成 draw.io 原生对象，将简单对象改成独立 SVG。

### 文件太大

压缩背景 PNG，减少局部 PNG 数量，复用 SVG，降低不必要的高分辨率位图，避免重复嵌入大素材。

## 交付用语模板

```text
已生成可编辑 draw.io 文件，并打包了背景 PNG、SVG 元素、manifest 清单和预览图。

- draw.io 源文件：project_name.drawio
- 完整素材包：project_name_drawio_package.zip
- 背景：assets/background.png
- 素材：assets/*
- 使用说明：README_使用说明.md
```
